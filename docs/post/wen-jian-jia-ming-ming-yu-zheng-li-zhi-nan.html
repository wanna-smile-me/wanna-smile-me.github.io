文件夹命名与整理指南

一、文件夹层级结构

遵循三层封顶原则，避免文件夹层次过多，寻找麻烦

第一层：大类

核心分类，涵盖主要使用场景，例如：项目工作、日常办公、培训学习、娱乐等。

第二层：小类

对应具体项目、工作内容、会议等

第三层：项目文件夹

针对具体项目或任务的文件夹，可根据工作流程进一步细分，无需额外增加层级。

下面是我常用建筑设计流程文件夹示例：

📁01_Reference1（前置任务）

📁01_Reference2(参考资料等)

📁02_Working1（工作文件）

📁02_Working2(其他不同类型工作文件)

📁03_Output（导出结果）

这里通过前置数字作为分类不同工作流程文件而非一个类别文件夹，避免层级过多

二、命名格式

1. 统一格式原则

统一采用标志优先級 + 核心內容 + 日期的格式，不使用过深的嵌套层级。

例如4.2_建筑设计九_20251006

01_工作照片_241102

2. 标志优先级规范

标志优先级用于区分文件/文件夹的重要程度、使用顺序或流程阶段，可以采用数字、字母和标点来标注，根据个人使用习惯和需求可以灵活命名和使用倒叙与正序排列（比如我本人大学期间学校课程文件用4.1 代表第四学年第一学期，使用倒叙排列使最新文件排在最前面）

01 01 02 02 03

1.1 1.2 2.1

4.2 (4.1) (3.1)

z_01 z_02 z_03

3. 核心内容规范

核心内容应该简洁明了，直接体现文件或文件夹的用途，避免冗余和比较模糊表述，也不能过分精简得让自己无法分辨

4. 日期格式规范

优先使用阿拉伯数字，确保排序清晰（系统默认按数字顺序排列），避免中文数字混淆

年度：YYYY （如2026）

年月：YYYYMM YYYY_MM YYYY-MM 如（202601 2026-01）

年月日：YYMMDD YYYYMMDD YY-MM-DD （如260101 20260101 2026-01-01 ）

注意：月份、日期不足2位补0（如01月、03日，避免出现「2026-5-2」这类不规范格式）。

5. 归档文件

个人喜欢使用通过 增加文件夹前缀 对归档文件进行标注，而非创建一个''已归档''文件夹，避免文件夹层级过多

根据不同场景需要文件以倒序或正序两种顺序排列文件夹，因此为了保持已归档的文件夹显示在底部，同时不破坏原有排列顺序，我习惯用两种前缀来标注归档文件

倒叙排列:

用括号(也可使用其他标点作为前缀)扩起标志优先级数字，使已归档文件显示在底部

📁4.1_xxxx_xxxx（正在使用的高阶优先级文件）

📁3.2_xxxx_xxxx(正在使用的文件)

📁(3.2)_xxxx_xxxx

📁(3.1)_xxxx_xxxx（已归档文件）

正序排列：

在标志优先级数字前加上字母如*_z*等，使已归档文件显示在底部

📁01_xxxx_xxxx（正在使用的高阶优先级文件）

📁02_xxxx_xxxx(正在使用的文件)

📁z_01_xxxx_xxxx

📁z_02_xxxx_xxxx（已归档文件）

6. 临时文件

单独建立「临时文件」文件夹，存放短期使用（1周内）的文件，定期清理，避免杂乱

三、文件整理

  1. 合理分类
  2. 文件命名与所在文件夹命名逻辑一致
  3. 及时清理：定期删除无用文件（如重复版本、过期临时文件），避免占用空间、干扰检索
  4. 同一文件的不同修改版本，可在核心内容后添加版本或日期标识（如03-建筑设计方案-20260101 03建筑设计方案-20260105）

四、方便后续维护

  1. 不使用模糊命名：禁止「新建文件夹1」「文档1」「资料」等无意义命名，避免后续无法识别；
  2. 及时归档文件，完成工作任务后，立即按归档规则标注归档，避免未归档文件堆积，便于后续查找历史文件；
  3. 命名中尽量减少中文与英文标点混用情况，推荐建议统一为英文标点；
  4. 定期维护：每月花10-15分钟检查文件夹命名规范、清理无用文件、整理归档，清理临时文件夹。
